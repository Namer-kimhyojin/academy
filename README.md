# 배터리아카데미 교육 수요조사 서비스

이차전지 분야 교육생(취업준비생·직무전환자)을 대상으로 한 **웹 설문 + 실시간 통계 분석 대시보드**입니다.
학생이 웹에서 설문에 응답하면 서버에 자동 취합되고, 운영자는 비밀번호로 보호된 대시보드에서 통계를 바로 확인할 수 있습니다.

외부 라이브러리가 전혀 없어(순수 Node.js + Docker) **Synology NAS의 Container Manager에서 그대로 구동**됩니다.

---

## 구성

| 파일 | 설명 |
|------|------|
| `server.js` | 응답 저장·조회 API + 정적 파일 서버 (의존성 없음) |
| `public/index.html` · `survey.js` | 학생용 설문 페이지 |
| `public/admin.html` · `admin.js` | 운영자용 통계 분석 대시보드 |
| `public/questions.js` | **설문 문항 정의(공유)** — 여기만 고치면 설문·대시보드가 함께 바뀜 |
| `public/styles.css` | 공통 디자인 |
| `Dockerfile` · `docker-compose.yml` | 배포용 |
| `data/` | 응답 데이터(`responses.ndjson`)가 쌓이는 폴더 |

### 설문 구성 (11개 섹션 · 오프라인 교육 기준)

1. 기본 정보 (연령·**성별·거주지역**·학력·전공·경험 등 — 그룹 비교용) *※ 기수는 묻지 않고 접속 링크로 자동 구분*
2. 사전 지식·역량 자가진단 (10개 항목 5점 척도 → 역량 레이더)
3. 관심 교육 주제 (관심도 + 우선순위 TOP3 + 희망 난이도)
4. 교육 방식 선호 (이론/실습 비중, 학습활동, 1회 수업 길이)
5. 교육 일정·시수 (참여 가능 시기/월, 주당 가능 시간, 선호 운영 형태) *※ 교육시간 수치는 설문에 노출하지 않음*
6. 학습 환경·인프라 (실습장비·통학시간·교재 형태 — 오프라인 기준)
7. **교통·숙박 지원** (이동 부담·셔틀버스 수요/노선·숙박 필요·숙박 개인부담 한도)
8. **취업지원 전문가 멘토링 수요** (참여의향·필요 항목·멘토 유형·방식·빈도)
9. 취업연계 서비스·추가 교육 프로그램 (채용연계·인턴·자격지원·부가교육)
10. 소통·네트워킹 (공지 채널·네트워킹 필요도·커뮤니티·스터디·건의 방식)
11. 진로·목표·동기 (목표 직무·기업·**취업 희망지역·희망 연봉**·동기 + 서술형)

## Synology NAS 설치 (Container Manager)

> DSM 7.2 이상 + Container Manager(구 Docker) 패키지가 설치되어 있어야 합니다.

### 1) 파일 올리기
File Station에서 `docker` 공유폴더 안에 `battery-survey` 폴더를 만들고,
이 프로젝트 폴더(`battery-academy-survey`) 안의 **모든 파일을 업로드**합니다.
경로 예시: `/docker/battery-survey/`

### 2) 비밀번호 변경 (중요)
`docker-compose.yml`을 텍스트 편집기로 열어
`ADMIN_PASSWORD: "1234"` 의 값을 **본인만 아는 비밀번호**로 바꿔 저장하세요. (현재 임시값은 `1234`)
이 비밀번호가 분석 대시보드 접속 암호입니다.

필요하면 포트도 바꿀 수 있습니다. 기본은 NAS의 `9080` 포트입니다. (`"9080:3000"`)

### 3) 프로젝트 생성
Container Manager → **프로젝트(Project)** → **생성**
- 프로젝트 이름: `battery-survey`
- 경로: 1)에서 만든 폴더 선택 (`/docker/battery-survey`)
- 소스: **기존 docker-compose.yml 사용** 선택
- **다음 → 완료**를 누르면 이미지를 빌드하고 컨테이너가 자동 실행됩니다.

### 4) 접속
같은 네트워크의 브라우저에서:
- 설문(학생용): `http://<NAS-IP>:9080/`
- 대시보드(운영자): `http://<NAS-IP>:9080/admin.html`

### 5) 외부(인터넷)에서 학생들이 응답하게 하려면
다음 중 하나로 외부 접속을 열어 주세요. **HTTPS(역방향 프록시) 사용을 권장합니다.**
- Synology **역방향 프록시**(제어판 → 로그인 포털 → 고급 → 역방향 프록시)로
  `survey.내도메인.com` → `localhost:9080` 연결 + Let's Encrypt 인증서 적용
- 또는 라우터 포트포워딩 + DDNS

## 일반 PC/서버에서 실행 (테스트용)

Node.js 18+ 가 설치되어 있으면 실행됩니다. 파일 저장 모드는 외부 패키지 없이 동작하고,
DB 모드로 테스트하려면 먼저 `npm install`(pg 설치) 후 `DATABASE_URL`을 지정하세요.

```bash
cd battery-academy-survey
ADMIN_PASSWORD=mypassword node server.js
# 설문:     http://localhost:3000/
# 대시보드: http://localhost:3000/admin.html
```

Windows(PowerShell):
```powershell
$env:ADMIN_PASSWORD="mypassword"; node server.js
```

## 운영자 사용법

- **대시보드 접속**: `/admin.html` → 비밀번호 입력
- **기수·연도 필터**: 상단에서 특정 기수/연도만 골라 그 그룹의 상세 통계만 볼 수 있음
- **기수별·연도별 비교**: 기수별/연도별 응답 수와, 선택한 지표(역량·관심도·멘토링 필요도·취업연계 필요도·참여희망률·실습비중)를 나란히 비교 → 기수가 거듭될수록 어떻게 달라지는지 추이 확인
- 실시간 응답 수, 평균 소요시간, 멘토링 참여 희망률(KPI)
- 섹션별 통계: 막대그래프, 역량 **레이더 차트**, 주제 **우선순위 가중점수**, 서술형 응답 목록
- **교차 분석**: "전공 계열별 사전 역량 평균"처럼 그룹 기준 × 지표를 골라 비교
- **CSV 내보내기**: 한글 깨짐 없는 엑셀용 CSV로 전체 원본 다운로드 (기수·연도 컬럼 포함, `⬇ 전체 CSV 내보내기`)
- `↻ 새로고침`으로 최신 응답 반영

## 기수별 접근 주소 · 연차(연도) 구분

설문에는 기수를 묻는 문항이 없습니다. 대신 같은 서버 주소에 **쿼리 파라미터**만 붙인
**기수별 전용 링크**로 기수·연도가 자동 지정됩니다. 학생은 받은 링크만 누르면 됩니다.

| 안내할 대상 | 공유할 주소 예시 |
|------------|-----------------|
| 3기 (2026년) | `http://<주소>:9080/?c=3기&y=2026` |
| 2기 (2026년) | `http://<주소>:9080/?c=2기&y=2026` |
| 1기 (2025년) | `http://<주소>:9080/?c=1기&y=2025` |

- `c=` 기수 (예: `3기` 또는 숫자 `3`), `y=` 연도(연차, 4자리). `y`를 생략하면 제출 시점의 연도로 자동 기록됩니다.
- 새 기수가 생기면 `public/questions.js`의 `SURVEY_COHORTS` 목록에 한 줄만 추가하세요. (예: `["4기","3기","2기","1기"]`) — 링크의 `c=` 값이 이 목록과 대조되어 정규화됩니다.
- 기수 파라미터(`c=`) 없이 기본 주소로 접속하면, 시작 화면에 "기수 정보 없는 링크" 경고가 표시되고 응답은 `(미기재)`로 분류됩니다. 반드시 전용 링크를 배포하세요.
- 모든 응답에는 기수와 연도가 함께 저장되어, 대시보드에서 **기수별·연도별로 정리·비교**됩니다.

## 인터넷에 무료로 올리기 (Render + Neon 무료 Postgres)

NAS가 없거나 외부에서 바로 접속하게 하고 싶다면, **Render(무료 Node 호스팅)** 에 올리고
**Neon(무료 Postgres)** 에 응답을 저장하면 됩니다. Render 무료 인스턴스는 일정 시간 미사용 시
잠들지만, **데이터는 Neon DB에 보존**되므로 잠들었다 깨어나도 응답이 사라지지 않습니다.

> 이 프로젝트는 `DATABASE_URL` 환경변수가 있으면 자동으로 Postgres에 저장하고,
> 없으면 기존처럼 파일(`data/responses.ndjson`)에 저장합니다. 따라서 NAS·로컬은 코드 수정 없이 그대로 동작합니다.

### 1) 코드를 GitHub에 올리기
이 폴더를 GitHub 저장소에 push 합니다. (`data/`와 `.env`는 `.gitignore`로 제외되어 안전합니다.)

### 2) Neon에서 무료 DB 만들기
1. <https://neon.tech> 가입 → **New Project** 생성
2. 대시보드의 **Connection string**(`postgresql://...?sslmode=require`)을 복사
   - 테이블(`responses`)은 서버가 처음 켜질 때 자동으로 생성하므로 따로 만들 필요 없습니다.

### 3) Render에 배포
1. <https://render.com> 가입 → **New +** → **Blueprint** 선택 후 GitHub 저장소 연결
   (저장소의 `render.yaml`을 자동 인식합니다. 또는 **New Web Service**로 수동 생성:
   Build Command `npm install`, Start Command `npm start`)
2. **Environment**에 환경변수 두 개 입력:
   - `DATABASE_URL` = 2)에서 복사한 Neon 연결 문자열
   - `ADMIN_PASSWORD` = 분석 대시보드 접속 비밀번호(직접 지정)
3. 배포가 끝나면 `https://<앱이름>.onrender.com` 주소가 생깁니다.
   - 설문: `https://<앱이름>.onrender.com/?c=3기&y=2026`
   - 대시보드: `https://<앱이름>.onrender.com/admin.html`

### 참고
- 무료 인스턴스는 약 15분 미사용 후 잠들어, 다음 접속 시 첫 응답이 수십 초 느릴 수 있습니다. (데이터 손실 아님)
- 항상 빠른 응답이 필요하면 Render 유료 플랜으로 올리거나 NAS 운영을 쓰세요.

## 코드(GitHub) 보관만 할 경우
- **GitHub Pages**는 정적 파일만 제공하므로 응답 수집 백엔드가 동작하지 않습니다. (설문 화면만 표시)
- 코드는 GitHub에 백업·버전관리하고, 실제 서비스는 위의 Render 또는 Synology NAS로 운영하는 것을 권장합니다.

## 데이터 관리

**Postgres 모드(Render/Neon)** — 권장
- 모든 응답이 Neon DB의 `responses` 테이블에 저장되어 재배포·재시작과 무관하게 영구 보존됩니다.
- 백업: Neon 콘솔에서 SQL 덤프를 받거나, 대시보드의 `⬇ 전체 CSV 내보내기`로 다운로드.

**파일 모드(NAS/로컬, `DATABASE_URL` 없음)**
- 모든 응답은 `data/responses.ndjson`에 **한 줄에 한 응답**(JSON)으로 누적됩니다.
- 이 폴더는 docker 볼륨으로 연결되어 컨테이너를 재시작·업데이트해도 보존됩니다.
- 백업은 `data/responses.ndjson` 파일만 복사하면 됩니다.
